Introduction

IBA_J was developped in order to 1) enable multiple data file conversion and 2) use the image analysis capabilities offered by ImageJ. The latter is the main reason why this plugin is developed under JAVA.

Current version is v2019-7-26, implementing this quick users guide.

 

Installation

To install the plugin, it is imperative to first install imageJ.

Then, unzip the plugin files in the App path\plugins directory keeping the files arborescence. The App path is the directory containing ij.jar file.

        App path\plugin\IBA_J\*.*

        App path\plugin\IBA_J*.class

        App path\IJ_prefs.txt

        App path\IJ_prop.txt

IBA_J Users guide

User interface and tools

Treatment of AIFIRA data requires 2 main steps:

        List files conversion (raw data *.lst) into individual detector files (*.pixe, *.rbs, etc..). Listfiles are generated during acquisition after saving all detected events (eg. Emitted particles, photons, scattered particles) in chronological order. In listfiles, events detected by different detectors are the mixed into one single file. This feature is interesting to manage coincidence and perform dynamic analysis. For imaging nevertheless, information corresponding to a single detector type is usually interesting.

        Mapping calculation and spatial ROI definition for a single detector type

List file conversion:

A)    Define list file structure and files output

Figure 2: overview of the conversion options window

 

Outpout files:

        Sorted listfiles: *.lst files are sorted according to ADC type definition and saved separately

o   RBS, STIM or PIXE types: separate list of (X,Y,E) events with file extension defined according to their type. No file header

o   SupaVisio option: only for compatibility (support of SupaVisio program has been stopped). Saves a list of (x,y,E) events with a specific supavisio header

        Gupix singles: export PIXE spectra files in Gupix format for PIXE ADCs separately

        Gupix Sum: export a sum spectrum for defined PIXE ADCs Gupix format

        SimNRA: export RBS spectra in compatible SimNRA format

        STIM-J: export STIM spectra in STIM-J format

        STIM tack: create a stack of STIM maps instead of separate maps

 

Calculate

        STIM map: enables STIM median maps calculation

        PIXE maps: not implemented

        Tomo PIXE proj.sum: option for specific experiments

 

Display

        Spectra: all converted spectra display in separate windows

        STIM stack: only if STIM map calculation enabled

 

B)    Select files and convert them according to options defined at step A.

 

Mapping calculation and spatial ROI definition for a single detector type

Figure 3: overview of the plugin windows used for mapping

 

Mapping and spatial ROI definition are made using sorted listfile as obtained in previous steps. To calculate the local spectra corresponding to any region of interest, just follow theses steps:

        Open a sorted list file (D), eg *.PIXE or *.RBS

        The plugin displays the total spectra (a) and pre-defined energy windows as recorded in the IJ_prefs.txt file (b). Saved ROI parameters are energy window (min, max), name and active/inactive state. The corresponding parameters are recorded in the following format in the IJ_prefs.txt file (example from fig. 3, Mg ROI):

o   .IBA.roi2.name = Mg

o   .IBA.roi2.min = 291

o   .IBA.roi2.max = 315

o   .IBA.roi2.isActive = true

o    

o   +ROI button: add a new set of buttons and text fields allowing to define a new ROI

        Once, all ROI have been define, press Generate pictures button in order to calculate energy maps (c).

        Using the drawing tools of ImageJ, define ROIs and add them to the imageJ ROI manager (d)

        In the map window, press the button calculate ROI spectra in order to proceed to local spectra calculation corresponding to ROI manager list.

        For each ROI, the plugin displays a new spectrum window (e) and automatically save the spectrum as a text file.
